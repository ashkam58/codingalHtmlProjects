Comments in HTML

The secret notes left by developers, for developers.

What are HTML Comments?

HTML comments are sections of code that are completely ignored by the web browser. They won't show up on the screen, but anyone looking at the raw code can read them!

They are written using the <!-- and --> syntax.

<!-- This is a comment. The user cannot see this on the webpage! -->
<p>This is a regular paragraph that the user CAN see.</p>

Why use comments?

Comments are incredibly useful for several reasons:

  • Documentation: Explain what complex parts of your code do so you (or someone else) can understand it later.
  • Debugging: Temporarily hide ("comment out") sections of code to see if they are causing errors, without actually deleting them.
  • Organization: Divide your HTML file into clear sections like <!-- Header Section --> or <!-- Footer Section -->.

Commenting Out Code

You can wrap actual HTML tags in a comment to temporarily hide them from the browser during testing or redesigns.

<!--
<div class="unfinished-feature">
    <p>This feature is not ready to be seen yet!</p>
</div>
-->
